GroovyEclipse Types Extensions

Identifier:
org.codehaus.groovy.eclipse.org.codehaus.groovy.eclipse.core.types

Since:
[Enter the first release in which this extension point appears.]

Description:
In order to adapt GroovyEclipse to specific domains, type extensions can be implemented. These extensions help describe the custom domain and also provide types for various domain specific classes. The end result is new types visible to GroovyEclipse with which to perform code completion, type inference etc.

Configuration Markup:

<!ELEMENT extension (memberLookup)+>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED

>


<!ELEMENT memberLookup EMPTY>

<!ATTLIST memberLookup

name             CDATA #IMPLIED

requiredContexts CDATA #IMPLIED

class            CDATA #REQUIRED

>

  • name - A name suitable for display.
  • requiredContexts - A comma separated list of contexts that this context depends on. For this member lookup to be selected, the source location must be in one of the contexts in the list.
  • class -
